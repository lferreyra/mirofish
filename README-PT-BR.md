## ⚡ Visão Geral

**MiroFish** é um motor de previsão de IA de próxima geração alimentado por tecnologia multi-agentes. Ao extrair informações de sementes do mundo real (como notícias de última hora, esboços de políticas, sinais financeiros), ele constrói automaticamente um mundo digital paralelo de alta fidelidade. Neste espaço, milhares de agentes inteligentes com personalidades independentes, memória de longo prazo e lógica comportamental interagem livremente e evoluem socialmente. Você pode injetar variáveis dinamicamente a partir de uma "visão de Deus" para deduzir com precisão as trajetórias futuras — **ensaie o futuro em uma caixa de areia digital e vença decisões após inúmeras simulações**.

> Tudo o que você precisa fazer é: Carregar materiais de sementes (relatórios de análise de dados ou histórias de ficção interessantes) e descrever suas necessidades de previsão em linguagem natural</br>
> MiroFish retornará: Um relatório de previsão detalhado e um mundo digital altamente interativo e de alta fidelidade

### Nossa Visão

MiroFish é dedicado a criar um espelho de inteligência coletiva que mapeia a realidade. Ao capturar a emergência coletiva desencadeada por interações individuais, quebramos as limitações da previsão tradicional:

- **Em Nível Macro**: Somos um laboratório de ensaio para tomadores de decisão, permitindo que políticas e relações públicas sejam testadas com zero risco
- **Em Nível Micro**: Somos uma caixa de areia criativa para usuários individuais — seja deduzindo finais de romances ou explorando cenários imaginativos, tudo pode ser divertido, lúdico e acessível

De previsões sérias a simulações lúdicas, deixamos cada "e se" ver seu resultado, tornando possível prever qualquer coisa.

## 🔄 Fluxo de Trabalho

1. **Construção de Gráfico**: Extração de sementes & Injeção de memória individual/coletiva & Construção GraphRAG
2. **Configuração do Ambiente**: Extração de relações de entidades & Geração de personas & Injeção de configuração de agente
3. **Simulação**: Simulação paralela de plataforma dupla & Análise automática de requisitos de previsão & Atualizações dinâmicas de memória temporal
4. **Geração de Relatório**: ReportAgent com conjunto rico de ferramentas para interação profunda com ambiente pós-simulação
5. **Interação Profunda**: Converse com qualquer agente no mundo simulado & Interaja com ReportAgent

## 🚀 Início Rápido

### Opção 1: Implantação por Código-Fonte (Recomendado)

#### Pré-requisitos

| Ferramenta | Versão | Descrição | Verificar Instalação |
|------|---------|-------------|-------------------|
| **Node.js** | 18+ | Ambiente de execução de frontend, inclui npm | `node -v` |
| **Python** | ≥3.11, ≤3.12 | Ambiente de execução de backend | `python --version` |
| **uv** | Última | Gerenciador de pacotes Python | `uv --version` |

#### 1. Configurar Variáveis de Ambiente

```bash
# Copie o arquivo de configuração de exemplo
cp .env.example .env

# Edite o arquivo .env e preencha as chaves de API necessárias
```

**Variáveis de Ambiente Necessárias:**

```env
# Configuração de API LLM (suporta qualquer API LLM com formato OpenAI SDK)
# Recomendado: Modelo Alibaba Qwen-plus via Plataforma Bailian: https://bailian.console.aliyun.com/
# Alto consumo, tente simulações com menos de 40 rodadas primeiro
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Configuração Zep Cloud
# Cota mensal gratuita é suficiente para uso simples: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. Instalar Dependências

```bash
# Instalação única de todas as dependências (raiz + frontend + backend)
npm run setup:all
```

Ou instale passo a passo:

```bash
# Instalar dependências de Node (raiz + frontend)
npm run setup

# Instalar dependências Python (backend, cria automaticamente ambiente virtual)
npm run setup:backend
```

#### 3. Iniciar Serviços

```bash
# Iniciar frontend e backend (execute a partir da raiz do projeto)
npm run dev
```

**URLs dos Serviços:**
- Frontend: `http://localhost:3000`
- API do Backend: `http://localhost:5001`

**Iniciar Individualmente:**

```bash
npm run backend   # Iniciar apenas o backend
npm run frontend  # Iniciar apenas o frontend
```

### Opção 2: Implantação com Docker

```bash
# 1. Configurar variáveis de ambiente (mesmo que implantação por código-fonte)
cp .env.example .env

# 2. Puxar imagem e iniciar
docker compose up -d
```

Lê `.env` do diretório raiz por padrão, mapeia portas `3000 (frontend) / 5001 (backend)`

> Endereço de espelho para puxar mais rapidamente é fornecido como comentários em `docker-compose.yml`, substitua se necessário.
