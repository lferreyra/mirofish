# Source Investigation List v1

Date: March 16, 2026

## Purpose

Preserve a broad candidate list of information sources to investigate for
MiroFish ingestion.

This list was generated externally from the source-priority criteria already
discussed for structural information arbitrage.

Important distinction:

- this document is a `candidate source backlog`
- it is **not** the canonical priority surface

Canonical prioritization still lives in:

- [Source Priority Matrix v1](/home/d/codex/MiroFish/docs/research-frameworks/2026-03-16-source-priority-matrix-v1.md)

This file should be used as:

- a discovery backlog
- a venue list to validate
- a set of targets for future ingestion connectors, crawlers, and monitoring

It should **not** be treated as fully vetted.

## Usage note

Before any source from this list becomes first-class in MiroFish, it should be
checked for:

- actual relevance to our target domains
- practical accessibility
- signal quality
- repeatability of ingestion
- whether it is upstream enough to matter before market awareness catches up

## Best connected-node sources

| Source | Key connections |
| --- | --- |
| `Federal Register / BIS / Commerce` | Rule changes impact miners, refiners, OEMs, hyperscalers, and export-dependent suppliers. |
| `IEA / SIA / SEMI` | Reports connect supply concentration, demand growth, lead times, and policy response. |
| `SEC 10-K / 10-Q / earnings transcripts` | Connect customer concentration, capacity, debt, geography, and risk. |
| `DoD / DOE procurement awards` | Reveal state-prioritized bottlenecks across AI, energy, defense, and infrastructure. |
| `IEEE / JEDEC standards` | Architecture and standards shifts can reorder dependency graphs. |
