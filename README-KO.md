## ⚡ 프로젝트 개요

**MiroFish**는 멀티 에이전트 기술 기반의 차세대 AI 예측 엔진입니다. 현실 세계의 시드 정보(속보, 정책 초안, 금융 시그널 등)를 추출하여 고충실도의 평행 디지털 세계를 자동 구축합니다. 이 공간 안에서 독립적인 인격, 장기 기억, 행동 로직을 갖춘 수많은 지능형 에이전트가 자유롭게 상호작용하며 사회적 진화를 거칩니다. 신의 관점에서 변수를 동적으로 주입하여 미래의 흐름을 정밀하게 추론할 수 있습니다 — **즉, 디지털 샌드박스에서 미래를 리허설하고, 수많은 시뮬레이션 끝에 최선의 결정을 내릴 수 있게 해줍니다**.

> 준비물: 시드 자료(데이터 분석 보고서, 흥미로운 소설 등)를 업로드하고, 자연어로 예측 요구사항을 설명하기만 하면 됩니다</br>
> MiroFish가 돌려주는 것: 상세한 예측 보고서와 심층 상호작용이 가능한 고충실도 디지털 세계

### 우리의 비전

MiroFish는 현실을 비추는 군집 지능의 거울을 만드는 것을 목표로 합니다. 개체 간 상호작용이 촉발하는 집단 창발 현상을 포착하여 기존 예측의 한계를 뛰어넘습니다:

- **거시적 관점**: 의사결정자를 위한 리허설 실험실 — 정책과 PR을 제로 리스크로 테스트
- **미시적 관점**: 개인 사용자를 위한 크리에이티브 샌드박스 — 소설 결말 추론부터 상상 시나리오 탐색까지, 재미있고 직관적으로

진지한 예측부터 재미있는 시뮬레이션까지, 모든 "만약에?"의 결과를 볼 수 있게 합니다.

1. **그래프 구축**: 현실 시드 추출 & 개체/집단 기억 주입 & GraphRAG 구축

3. **시뮬레이션**: 듀얼 플랫폼 병렬 시뮬레이션 & 예측 요구사항 자동 분석 & 실시간 기억 업데이트

5. **심층 대화**: 시뮬레이션 세계 속 누구와도 1:1 대화 & ReportAgent와 대화

## 🚀 빠른 시작

### 방법 1: 소스 코드 배포 (권장)

#### 사전 요구사항

| 도구 | 버전 | 설명 | 설치 확인 |
|------|------|------|----------|
| **Node.js** | 18+ | 프론트엔드 런타임, npm 포함 | `node -v` |
| **Python** | ≥3.11, ≤3.12 | 백엔드 런타임 | `python --version` |
| **uv** | 최신 | Python 패키지 매니저 | `uv --version` |

#### 1. 환경 변수 설정

```bash
# 예시 설정 파일 복사
cp .env.example .env

# .env 파일을 편집하여 필요한 API 키 입력
```

**필수 환경 변수:**

```env
# LLM API 설정 (OpenAI SDK 형식을 지원하는 모든 LLM API 사용 가능)
# 권장: 알리바바 바이롄 플랫폼의 qwen-plus 모델: https://bailian.console.aliyun.com/
# 소비가 크므로, 처음에는 40라운드 미만의 시뮬레이션부터 시도하세요
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Zep Cloud 설정
# 월간 무료 크레딧으로 간단한 사용 가능: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. 의존성 설치

```bash
# 모든 의존성 한번에 설치 (루트 + 프론트엔드 + 백엔드)
npm run setup:all
```

또는 단계별 설치:

```bash
# Node 의존성 설치 (루트 + 프론트엔드)
npm run setup

# Python 의존성 설치 (백엔드, 가상환경 자동 생성)
npm run setup:backend
```

#### 3. 서비스 시작

```bash
# 프론트엔드와 백엔드 동시 시작 (프로젝트 루트에서 실행)
npm run dev
```

**서비스 URL:**
- 프론트엔드: `http://localhost:3000`
- 백엔드 API: `http://localhost:5001`

**개별 시작:**

```bash
npm run backend   # 백엔드만 시작
npm run frontend  # 프론트엔드만 시작
```

### 방법 2: Docker 배포

```bash
# 1. 환경 변수 설정 (소스 배포와 동일)
cp .env.example .env

# 2. 이미지 풀 및 시작
docker compose up -d
```

기본적으로 루트 디렉토리의 `.env`를 읽으며, 포트 `3000 (프론트엔드) / 5001 (백엔드)`를 매핑합니다.

> 빠른 풀링을 위한 미러 주소가 `docker-compose.yml`에 주석으로 제공됩니다. 필요시 교체하세요.
