## Spacing Scale

Standard 8-point scale. MiroFish uses no spacing tokens currently — this scale becomes the authoritative reference for Phase 2 component audit.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Button horizontal padding, compact element spacing |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding, panel gaps |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical rhythm |

Exceptions:
- Touch targets (buttons) minimum 48px height — applies to all `.action-btn`, `.tab-pill`, and `.start-engine-btn` elements
- Step number circles: 32px diameter (existing pattern — keep)

## Color

All tokens use CSS `hsl()` function syntax (not space-separated Tailwind channels).
Token definitions live in `frontend/src/style.css`, imported in `frontend/src/main.js`.

### Token Definitions

```css
:root {
  --background:           hsl(210, 57%, 11%);   /* dark navy — page background */
  --foreground:           hsl(220, 33%, 97%);   /* near-white — body text */
  --primary:              hsl(217, 75%, 47%);   /* blue — CTAs, active states */
  --primary-foreground:   hsl(0, 0%, 100%);     /* white — text on primary */
  --secondary:            hsl(213, 40%, 18%);   /* mid-navy — card backgrounds */
  --secondary-foreground: hsl(220, 33%, 97%);   /* near-white — text on secondary */
  --accent:               hsl(213, 75%, 60%);   /* lighter blue — highlights, hover, success */
  --accent-foreground:    hsl(0, 0%, 100%);     /* white — text on accent */
  --muted-foreground:     hsl(213, 20%, 60%);   /* desaturated blue-gray — secondary text */
  --border:               hsl(213, 30%, 22%);   /* dark blue-gray — borders, dividers */
  --card:                 hsl(213, 50%, 14%);   /* deepest navy — elevated card surfaces */
  --destructive:          hsl(0, 84%, 60%);     /* red — destructive actions only */
  --destructive-foreground: hsl(0, 0%, 100%);  /* white — text on destructive */
}
```

**Source:** REQUIREMENTS.md R2.1 (7 tokens), RESEARCH.md Code Examples (added --card, --primary-foreground, --secondary-foreground, --accent-foreground, --destructive from Slater Consulting SaaS index.css)

### 60/30/10 Distribution

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--background` | hsl(210, 57%, 11%) | Page background, main content area, `#app` background |
| Secondary (30%) | `--secondary` + `--card` | hsl(213, 40%, 18%) / hsl(213, 50%, 14%) | Step cards, panels, sidebars, modals, action bars, workbench panels |
| Accent (10%) | `--primary` + `--accent` | hsl(217, 75%, 47%) / hsl(213, 75%, 60%) | See reserved list below |
| Destructive | `--destructive` | hsl(0, 84%, 60%) | Destructive actions only (none identified in Phase 2 scope) |

**Accent (`--primary`) reserved for:**
- Primary action buttons (`.action-btn`, `.start-engine-btn`)
- Active step card border (`border-color` on `.step-card.active`)
- Processing state dot/spinner (`.status-indicator.processing .dot`)
- Progress bar fill
- Active tab pill (`.tab-pill.active`) background
- Hovered link text color

**Accent (`--accent`) reserved for:**
- Hover state on primary buttons (lighten on hover)
- Completed state badges — replaces all `#10B981` / `#1A936F` / `#4CAF50` / `#E8F5E9` / `#2E7D32` / `#047857` green instances
- Active section highlight in Step5Interaction report sections
- Status icon "available" color in HistoryDatabase (replaces `#3B82F6` for graph build icon and `#10B981` for report icon; `#F59E0B` amber for env setup is replaced by `--accent` too)
- Selected/hovered link in D3 graph (replaces `#3498db` highlight stroke)

### Semantic Green Replacement Decision

**Decision:** Replace ALL green success indicators with `--accent` (`hsl(213, 75%, 60%)`).
**Rationale:** REQUIREMENTS.md R2.4 states "no red/green clashing with theme." The brand is dark navy/blue — green semantic colors are visually dissonant.
**Specific replacements:**
- `#10B981`, `#1A936F`, `#4CAF50`, `#2E7D32`, `#047857`, `#065F46`, `#1A936F` → `var(--accent)` for text/border
- `#ECFDF5`, `#D1FAE5`, `#E8F5E9` green badge backgrounds → `var(--secondary)` with `var(--accent)` text
- `#E9724C` (orange in D3 palette) → replace in D3 array (see D3 section below)
